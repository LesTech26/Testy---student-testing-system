## 2. Стек технологий (Technology Stack)

- **Backend:** PHP (без использования фреймворка, процедурный стиль).
- **База данных:** MySQL.
- **Frontend:** HTML5, CSS3, JavaScript (нативный).
- **Библиотеки и фреймворки:**
  - Bootstrap 5 (для верстки и адаптивности).
  - Bootstrap Icons (для иконок).
- **Верстка:** Адаптивная (responsive), ориентирована на мобильные устройства и десктопы.

## 3. Архитектура системы

Система построена по классической клиент-серверной архитектуре и не использует MVC-фреймворк. Логика разделена по функциональному принципу и ролям пользователей в файловой структуре.

### 3.1. Основные компоненты

1. **Модуль авторизации:** Раздельные точки входа для студентов (`index.php`), преподавателей (`teacher/login.php`) и администраторов (`admin/login.php`).
2. **Студенческая часть (Student Interface):**
   - Просмотр списка доступных тестов с фильтрацией.
   - Прохождение теста в защищенном режиме.
   - Просмотр результатов и детализированного разбора ответов.
3. **Преподавательская часть (Teacher Interface):**
   - Панель управления с общей статистикой.
   - Управление тестами (создание, редактирование, удаление, активация/деактивация).
   - Настройка системы оценивания (ручная с настраиваемыми правилами или автоматическая).
   - Управление вопросами (добавление, редактирование, удаление, загрузка изображений).
   - Просмотр результатов тестирования студентов с возможностью фильтрации и редактирования оценок.
   - Просмотр списка студентов с фильтрацией по курсу, специальности и городу.
4. **Административная часть (Admin Interface):**
   - **Супер-администратор:** Полный доступ ко всем функциям, управление администраторами, городами, специальностями.
   - **Городской администратор:** Ограниченный доступ к данным своего города.

## 4. Структура базы данных (Основные сущности)

*(На основе анализа кода, полная схема может отличаться)*

- **`students`**: Студенты.
  - `id`, `last_name`, `first_name`, `course_number`, `username`, `password`, `city`, `specialty`, `last_login`, `created_at`, `updated_at`.
- **`teachers`**: Преподаватели.
  - `id`, `last_name`, `first_name`, `username`, `password`, `city`, `is_active`, `created_at`.
- **`admins`**: Администраторы.
  - `id`, `last_name`, `first_name`, `username`, `password`, `city`, `is_superadmin`, `is_active`, `created_at`.
- **`tests`**: Тесты.
  - `id`, `title`, `description`, `course_id`, `city`, `specialty_id`, `time_limit`, `passing_score`, `grading_system` (`manual`/`auto`), `grading_rules` (JSON), `is_active`, `created_at`.
- **`questions`**: Вопросы теста.
  - `id`, `test_id`, `question_text`, `question_image`, `question_type` (`multiple_choice`), `option_a`, `option_b`, `option_c`, `option_d`, `correct_answer`, `explanation`.
- **`test_results`**: Результаты попыток тестирования.
  - `id`, `student_id`, `test_id`, `score`, `total_questions`, `percent`, `grade`, `is_passed`, `time_spent`, `completed_at`.
- **`student_answers`**: Детальные ответы студента на каждый вопрос.
  - `id`, `student_id`, `test_id`, `question_id`, `student_answer` (буква или текст), `is_correct`.
- **`specialties`**: Специальности.
  - `id`, `name`, `code`, `description`, `created_at`.

## 6. Детальное описание ключевых модулей

### 6.1. Авторизация (`index.php`, `login.php`, `teacher/login.php`, `admin/login.php`)

- Поддерживается вход для трех ролей: **Студент**, **Преподаватель**, **Администратор**.
- Для каждой роли своя форма входа, которая отображается динамически с помощью JavaScript.
- Формы отправляют данные на разные обработчики:
  - Студент → `login.php`
  - Преподаватель → `teacher/login.php`
  - Администратор → `admin/login.php`
- Реализована функция "Запомнить меня", которая сохраняет логин в `localStorage` браузера.
- Преподаватели и администраторы имеют привязку к городу (`city`), что ограничивает их доступ к данным.

### 6.2. Студенческая панель: Выбор теста (`tests.php`)

- Отображает список тестов, доступных студенту, с фильтрацией по:
  - **Курсу:** Только тесты для курса студента или общие.
  - **Городу:** Только тесты для города студента или общие.
  - **Специальности:** Только тесты для специальности студента или общие.
- Для каждого теста отображается:
  - Статус: доступен, пройден/не пройден.
  - Основная информация: название, описание, время, проходной балл.
  - Если тест пройден, показывается полученная оценка и процент выполнения.
- Содержит блок с правилами прохождения тестирования, описывающий систему безопасности и последствия нарушений.

### 6.3. Модуль тестирования и защиты (`test.php`)

Это самый сложный и важный компонент системы.

**Функциональность:**
- Отображает вопросы по одному.
- Поддерживает типы вопросов: одиночный выбор (A, B, C, D) и текстовый ответ.
- Отображает изображение к вопросу (если есть) с возможностью открытия в модальном окне.
- **Таймер:** Ведет обратный отсчет времени, отведенного на тест. При истечении времени тест принудительно завершается.
- **Индикатор прогресса:** Показывает, сколько вопросов отвечено, и общий прогресс в виде точек.

**Система защиты от списывания:**
Код реализует многоуровневую защиту:
1. **Запрет на переключение вкладок (`visibilitychange`):** Если студент переключился на другую вкладку более чем на 1 секунду, это фиксируется как нарушение.
2. **Запрет на переключение окон (`blur/focus`):** Аналогично, уход из окна браузера фиксируется как нарушение.
3. **Блокировка Print Screen и комбинаций клавиш:**
   - Клавиша `PrintScreen`.
   - Комбинация `Win+Shift+S` (инструмент "Набросок на фрагменте экрана" в Windows).
   - Комбинации для создания скриншотов на Mac (`Cmd+Shift+3/4/5`).
   - `Ctrl+P` (печать).
   - `F12`, `Ctrl+Shift+I`, `Ctrl+Shift+J`, `Ctrl+Shift+C` (инструменты разработчика).
4. **Блокировка контекстного меню и копирования:** Запрещены `contextmenu`, `copy`, `cut`, `paste`.
5. **Механизм нарушений:**
   - За каждое нарушение увеличивается счетчик `screenshot_attempts` в сессии (`update_screenshot_counter.php`).
   - При первом нарушении — всплывающее предупреждение.
   - При втором нарушении тест автоматически завершается, и студенту выставляется оценка "2". Это реализовано через функцию `autoFailTest()`.

### 6.4. Результаты теста (`result.php`)

- Отображает итоговую статистику: количество правильных ответов, процент, оценку, затраченное время.
- Указывает, сдан тест или нет.
- Предоставляет детальный разбор каждого вопроса:
  - Текст вопроса.
  - Ваш ответ (текст или буква варианта).
  - Правильный ответ.
  - Объяснение (если есть).

## 7. Преподавательская часть (Teacher Interface)

### 7.1. Панель управления преподавателя (`teacher/index.php`)

- Отображает общую статистику по тестам, вопросам, студентам и результатам с учетом города преподавателя.
- Показывает последние активности (результаты тестирования).
- Предоставляет быстрые действия для создания теста, добавления вопросов, просмотра студентов и результатов.
- Отображает распределение студентов по городам.

### 7.2. Управление тестами (`teacher/tests.php`)

- **CRUD операции:** Создание, просмотр, редактирование, удаление, активация/деактивация тестов.
- **Привязка к городу:** Тесты создаются с привязкой к городу преподавателя (если указан).
- **Фильтрация:** Преподаватель видит только свои тесты (из своего города) и общие тесты.
- **Настройки теста:**
  - Время на прохождение.
  - Привязка к курсу и специальности.
  - **Система оценивания:**
    - **Автоматическая:** Оценка рассчитывается системой.
    - **Ручная:** Преподаватель настраивает правила оценивания (гибкая система с JSON-правилами).
- **Интерфейс настройки правил оценивания:** Визуальный конструктор с предпросмотром, валидацией и динамическим добавлением/удалением правил.

### 7.3. Управление вопросами (`teacher/questions.php`)

- **CRUD операции:** Создание, просмотр, редактирование, удаление вопросов.
- **Привязка к тесту:** Вопросы создаются для конкретного теста.
- **Тип вопроса:** Одиночный выбор (A, B, C, D).
- **Изображения:** Поддержка загрузки изображений к вопросам (JPG, PNG, GIF, WebP) с предпросмотром и удалением.
- **Валидация:** Проверка заполнения всех вариантов ответов и выбора правильного ответа.

### 7.4. Просмотр результатов (`teacher/results.php`)

- **Фильтрация:** По курсу, тесту, статусу (сдал/не сдал), специальности, городу студента.
- **Учет города:** Преподаватель видит только результаты тестов из своего города (или общих).
- **Редактирование оценки:** Возможность изменить оценку студента через модальное окно.
- **Удаление результатов:** Удаление результатов с подтверждением (только при наличии прав).
- **Статистика:** Общее количество результатов, количество сдавших/не сдавших, средний балл.
- **Детализация:** Прогресс-бар с процентом выполнения, цветовая индикация оценки.

### 7.5. Просмотр студентов (`teacher/students.php`)

- **Фильтрация:** По курсу, специальности, городу.
- **Учет города:** Преподаватель видит только студентов из своего города (или без города).
- **Информация о студенте:** ФИО, курс, специальность, город, логин, дата регистрации.
- **Переход к результатам:** Кнопка для просмотра всех результатов конкретного студента (`student_results.php`).

### 7.6. Результаты студента (`teacher/student_results.php`)

- Детальный просмотр всех результатов тестирования для выбранного студента.
- Отображение прогресса, оценки, статуса и даты прохождения по каждому тесту.

## 8. Административная часть (Admin Interface)

### 8.1. Система ролей администраторов

| Тип администратора | Права доступа |
| :--- | :--- |
| **Супер-администратор** | Полный доступ ко всем функциям. Управление администраторами, городами, специальностями. Видит все данные в системе. |
| **Городской администратор** | Ограниченный доступ к данным своего города. Может создавать студентов, преподавателей и тесты только для своего города. Не имеет доступа к управлению администраторами и специальностями. |

### 8.2. Авторизация администратора (`admin/login.php`)

- Форма входа с проверкой логина и пароля (пароль хранится в открытом виде).
- Проверка поля `is_active` — неактивные администраторы не могут войти.
- После входа определяется тип администратора (`is_superadmin`) и выполняется перенаправление:
  - Супер-админ → `index.php`
  - Городской админ → `index_city.php`

### 8.3. Панель супер-администратора (`admin/index.php`)

- Отображает общую статистику по системе: тесты, студенты, вопросы, результаты, специальности, преподаватели, администраторы.
- Формы для создания:
  - **Городского администратора** (с привязкой к городу).
  - **Студента** (с выбором курса и специальности).
  - **Преподавателя** (с привязкой к городу).
- Быстрые действия и навигация по всем разделам админ-панели.
- Отображение информации о только что созданных пользователях с возможностью копирования логина и пароля.

### 8.4. Панель городского администратора (`admin/index_city.php`)

- Отображает статистику только по своему городу.
- Формы для создания студентов и преподавателей (город подставляется автоматически).
- Заблокированные разделы (управление администраторами, специальностями) отображаются как недоступные.
- Информационное сообщение о текущем городе.

### 8.5. Управление администраторами (`admin/admins.php`)

- **Доступ:** Только для супер-администратора.
- **Функции:**
  - Просмотр всех администраторов.
  - Редактирование данных (ФИО, город, пароль, статус активности).
  - Активация/деактивация администратора.
  - Назначение/снятие статуса супер-администратора.
  - Удаление администратора (нельзя удалить себя или другого супер-админа).
- **Интерфейс:** Таблица с подсветкой своей учетной записи, модальные окна для редактирования и удаления.

### 8.6. Управление студентами (`admin/students.php`)

- **Фильтрация:** По курсу, специальности, городу.
- **Массовые действия:** Массовое обновление городов (только для супер-админа).
- **CRUD операции:**
  - Просмотр всех студентов с отображением города, курса, специальности.
  - Редактирование студента (через модальное окно) с возможностью изменения пароля.
  - Быстрое изменение города (отдельное модальное окно).
  - Удаление студента с подтверждением (удаляются также все результаты тестов).
- **Экспорт:** Экспорт отфильтрованного списка студентов в CSV с поддержкой кириллицы.
- **Навигация:** Кнопка перехода к результатам студента (`student_results.php`).

### 8.7. Управление преподавателями (`admin/teachers.php`)

- **CRUD операции:**
  - Просмотр всех преподавателей.
  - Редактирование данных (ФИО, город, пароль).
  - Активация/деактивация преподавателя.
  - Удаление преподавателя.
- **Интерфейс:** Таблица с возможностью копирования пароля, модальные окна для редактирования и удаления.

### 8.8. Управление специальностями (`admin/specialties.php`)

- **CRUD операции:** Создание, просмотр, редактирование, удаление специальностей.
- **Поля специальности:** Название, код (опционально), описание (опционально).
- **Проверка при удалении:** Нельзя удалить специальность, к которой привязаны тесты.

### 8.9. Управление тестами (`admin/tests.php`)

- **CRUD операции:** Создание, просмотр, редактирование, удаление, активация/деактивация тестов.
- **Фильтрация по городу:** Для супер-админа — отображение города теста, возможность выбора города при создании. Для городского админа — только тесты своего города.
- **Настройки теста:**
  - Время на прохождение.
  - Привязка к курсу, специальности и городу.
  - **Система оценивания:** Автоматическая или ручная с настраиваемыми правилами (JSON).
- **Интерфейс настройки правил оценивания:** Визуальный конструктор с предпросмотром и валидацией.

### 8.10. Управление вопросами (`admin/questions.php`)

- **CRUD операции:** Создание, просмотр, редактирование, удаление вопросов.
- **Привязка к тесту:** Вопросы создаются для конкретного теста.
- **Тип вопроса:** Одиночный выбор (A, B, C, D).
- **Изображения:** Поддержка загрузки изображений с предпросмотром и удалением.
- **Валидация:** Проверка заполнения всех вариантов ответов и выбора правильного ответа.

### 8.11. Просмотр результатов (`admin/results.php`)

- **Фильтрация:** По курсу, тесту, статусу (сдал/не сдал), специальности.
- **Массовые действия:**
  - Выбор нескольких результатов для массового удаления.
  - Экспорт отфильтрованных результатов в CSV.
  - Печать таблицы результатов.
- **Редактирование оценки:** Возможность изменить оценку через модальное окно.
- **Статистика по специальностям:** Детальная статистика по каждой специальности с прогресс-барами.
- **Интерфейс:** Таблица с чекбоксами, прогресс-барами, цветовой индикацией оценок.

### 8.12. Результаты студента (`admin/student_results.php`)

- Детальный просмотр всех результатов конкретного студента.
- **Статистика:** Средний результат, всего баллов, всего вопросов, процент сдач.
- **Отображение:** Карточки с прогресс-барами, оценками, статусами и временем прохождения.
- **Распределение по оценкам и успеваемость:** Детальная статистика с прогресс-баром.

## 9. Особенности реализации и заметки

### 9.1. Безопасность

- **Пароли хранятся в открытом виде:** В текущей реализации пароли студентов, преподавателей и администраторов хранятся и сравниваются в открытом виде. **Это критическая уязвимость.** Рекомендуется немедленно внедрить хеширование паролей (`password_hash()` / `password_verify()`).
- **CSRF-защита:** В некоторых формах (например, удаление результатов) используется CSRF-токен.
- **Проверка прав доступа:** Во всех административных файлах выполняется проверка `requireAdminLogin()`, которая перенаправляет неавторизованных пользователей на страницу входа. Для городских администраторов дополнительно проверяется соответствие города.

### 9.2. Городская изоляция данных

- Преподаватели и городские администраторы имеют поле `city`.
- Тесты могут иметь поле `city` (если `NULL` — тест доступен во всех городах).
- Студенты имеют поле `city`.
- При создании записей город подставляется автоматически для городских администраторов.
- При просмотре данных применяется фильтрация: пользователь видит только записи, где `city` совпадает с его городом или `city IS NULL`.

### 9.3. Система оценивания

- Ручная система оценивания с настраиваемыми правилами хранится в формате JSON в поле `grading_rules`.
- Правила проверяются сверху вниз (от большего количества правильных ответов к меньшему).
- Интерфейс настройки включает предпросмотр, валидацию и динамическое добавление/удаление правил.

### 9.4. Экспорт данных

- **CSV экспорт:** Реализован для студентов (`students.php`) и результатов (`results.php`).
- Добавляется BOM (Byte Order Mark) для корректного отображения кириллицы в Excel.
- В экспорт результатов включается информация о примененных фильтрах.

### 9.5. Управление сессиями

- Система активно использует `$_SESSION` для хранения состояния пользователя, данных текущего теста и результатов нарушений.
- При выходе из системы сессия полностью уничтожается (`session_destroy()`).

### 9.6. Обработка ошибок

- Код содержит блоки `try-catch` для обработки исключений PDO.
- Сообщения об ошибках сохраняются в сессию и отображаются на следующих страницах.
- Для отладки включен вывод ошибок (`error_reporting(E_ALL)`).

### 9.7. AJAX-запросы

- Используется `update_screenshot_counter.php` для асинхронного обновления счетчика нарушений.
- `get_text_answers.php` для получения текстовых ответов студентов (преподавательская часть).

### 9.8. Загрузка изображений

- Реализована загрузка изображений к вопросам.
- Поддерживаемые форматы: JPG, PNG, GIF, WebP.
- Максимальный размер файла: 2MB.
- Изображения сохраняются в директорию `assets/uploads/questions/` с уникальным именем.
- При удалении вопроса изображение также удаляется с сервера.

### 9.9. Генерация паролей

- Во всех формах создания пользователей есть кнопка "Сгенерировать пароль".
- Пароль генерируется из латинских букв и цифр (для администраторов добавляются спецсимволы).
- После генерации пароль отображается в поле ввода, его можно скопировать.

### 9.10. Автогенерация логина

- Для студентов и администраторов есть опция "Автогенерация логина".
- Логин создается на основе фамилии, имени и города со случайными цифрами.
- При включенной опции поле логина блокируется для ручного ввода.
